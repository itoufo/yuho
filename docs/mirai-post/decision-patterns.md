# MirAI-POST Decision Patterns

## 1. 結果→構造→世界観
- 先に結果を見せる
- その結果が再現可能である構造を示す
- 最後に思想・世界観を言語化する

## 2. 速さは才能ではなく設計の副作用
- 速度は目的ではなく指標
- 意思決定の速さと設計の明確さが速度を生む

## 3. 努力を否定せず、構造で摩耗を減らす
- 努力は必要だが、構造がないと摩耗する
- 仕組み化で再現性と継続性を作る

## 4. “社会不適合”は欠陥ではなく特性
- 合わないルールを変える
- AIで特性を扱いやすい形に変換する

## 5. コミュニティは速さではなく思考で集める
- 速度崇拝ではなく、構造の理解を目的にする
- 強いファンは世界観への共鳴で生まれる
